# Cron jobs

## Entendiendo la sintaxis
In a crontab line, there are 5 time fields followed by the command. Each field represents a specific unit of time.

Here is the breakdown of the cron string:

```text
 *     *     *     *     *    command to execute
 ┬     ┬     ┬     ┬     ┬
 │     │     │     │     │
 │     │     │     │     └─ Day of the week (0–6 or 1-7) (0 and 7 = Sunday)
 │     │     │     └─────── Month (1–12)
 │     │     └───────────── Day of the month (1–31)
 │     └─────────────────── Hour (0–23)
 └───────────────────────── Minute (0–59)
```

## Where are the days?

The 5th field (the last one before the command) controls the Day of the Week.

For your specific schedule (Mondays, Thursdays, and Saturdays), we use `1,4,6`:

* 1: Monday
* 4: Thursday
* 6: Saturday

## The full command breakdown:

`00 11 * * 1,4,6 echo "Test" >> /tmp/test.txt`

1. `00`: Run exactly at minute 0.
2. `11`: Run at 11 AM (24-hour clock).
3. `*`: Run every day of the month.
4. `*`: Run every month of the year.
5. `1,4,6`: Only run if today is Monday, Thursday, or Saturday.
6. `echo...`: This is the actual instruction the computer follows.

Quick Tip: You can also use names like `MON,THU,SAT` instead of numbers in most Ubuntu versions to make it easier to read!


# Una prueba para empezar

```25 12 * * 5 echo "Hello World" >> ~/tmp/cron-test.txt```

## How to set it up:

1. Open the terminal and access the crontab editor:
   ```bash
   crontab -e
   ```
2. Scroll to the bottom of the file and paste the line with the command
3. Save and exit (in Nano, press `Ctrl + O`, then `Enter`, and finally `Ctrl + X`).
